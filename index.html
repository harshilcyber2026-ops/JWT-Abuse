Description

The server issues JWT tokens signed with HS256. However, it accepts the alg field from the token itself — meaning if you set alg: "none", the server skips signature verification entirely.


Login as guest, decode the token, change both the algorithm to "none" and the role to "admin", strip the signature, and send it to /api/admin/dashboard.

How JWT Works

header . payload . signature
header — algorithm & token type (base64url)
payload — claims: username, role, iat (base64url)
signature — HMAC(header+payload, secret) — skipped if alg=none
⚠ VULNERABILITY If alg is set to "none" and the server trusts it, signature verification is skipped. A forged token with "role":"admin" will grant admin access.
